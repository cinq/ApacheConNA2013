Tomcat 8 and TomEE

These presentations were done by Mark Thomas from SpringSource

TomEE is Tomcat with JGroups

Difference in replication of the sessions:

  • delta manager: replicates every session to all nodes
  • backup manager: primary and a backup node picked randomly
Tomcat 8 and TomEE (cont.)

Tomcat parallel deployment allows the old version of the software to serve content as the new version of the application is being deployed on some nodes.

mod_pagespeed

Every trick for getting better performance is done with this.

  • Automatic Image Compression & Resizing
  • Minify CSS, JavaScript and HTML
  • Inline small images, CSS, and JavaScript
  • Cache Extension
  • Defer JavaScript
  • CSS/JavaScript Combining
mod_pagespeed (cont.)

Balance between speedy delivery (optimized pages) and maintainable code

Automated tools should be used.

Works well with CDN

Can we used for deliverying the application content faster or to hint on what could be done for faster delivery

Libraries that are behind the magic

mod_spdy

SPDY is a Google open network protocol available in:

Browsers:

  • Chrome
  • Firefox/SeaMonkey
  • Opera
  • Amazon Silk
mod_spdy (cont.)

Servers:

  • Apache Web Server (w/ mod_spdy)
  • nginx
  • Jetty
  • Node.js
mod_spdy (cont.)

Requires SSL/TLS

Compresses headers

Servers can push resources to clients

HTTP 2.0 work will use SPDY as its starting point

mod_spdy for Apache Web Server is a tricky implementation

For the Apache Web Server to be ready for HTTP/2.0 some serious rework will be needed.
